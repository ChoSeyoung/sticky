Create `lib/engine/darkMode.ts` with these exports:

1. **`hasDarkMediaQuery(html: string): boolean`**
   - Regex test: `/\(prefers-color-scheme\s*:\s*dark\)/i`

2. **`activateDarkMediaQueries(html: string): string`**
   - Replace `@media` conditions containing `prefers-color-scheme: dark` with `@media all {`
   - Replace `@media` conditions containing `prefers-color-scheme: light` with `@media not all {` (suppresses light-only rules when dark mode active, per RESEARCH.md open question 3)
   - Regex: `/@media\b([^{]*)\(prefers-color-scheme\s*:\s*dark\)([^{]*)\{/gi` → `'@media all {'`
   - Regex: `/@media\b([^{]*)\(prefers-color-scheme\s*:\s*light\)([^{]*)\{/gi` → `'@media not all {'`

3. **Color utility functions (internal, not exported):**
   - `hexToHsl(hex: string): [number, number, number] | null` — parse #rgb, #rrggbb to [h, s, l] with h in 0-360, s and l in 0-100
   - `rgbToHsl(r: number, g: number, b: number): [number, number, number]` — RGB 0-255 to HSL
   - `hslToHex(h: number, s: number, l: number): string` — back to #rrggbb
   - `invertColorPartial(value: string): string` — if HSL lightness > 85 → invert to (100 - lightness); if lightness < 15 → invert to (100 - lightness); else return unchanged
   - `invertColorFull(value: string): string` — always invert: lightness → (100 - lightness)
   - `parseColorValue(cssValue: string): string` — extract and transform color values from CSS declarations. Handle patterns: `#fff`, `#ffffff`, `rgb(r,g,b)`, `rgba(r,g,b,a)`. Skip named colors (`red`, `transparent`) and CSS variables (`var(--x)`).

4. **`applyAutoInversion(html: string, strategy: DarkModeStrategy): string`**
   - Import cheerio and DarkModeStrategy from types
   - If strategy is `'none'`, return html unchanged immediately
   - Load html with `cheerio.load(html, { decodeEntities: false })` (same pattern as applyClientRules.ts, include the @ts-expect-error comment)
   - Walk all elements with `[style]` attribute: parse style string, for each `background-color` or `background` or `color` declaration, apply the inversion function (partial or full based on strategy)
   - Walk all elements with `bgcolor` attribute: invert the value
   - Walk all elements with `color` attribute (HTML attribute, not CSS): invert the value
   - Walk all `<style>` blocks: use regex to find color/background-color declarations and invert their values
   - Also inject a dark background wrapper: add `body { background-color: #1a1a1a !important; }` as a `<style>` tag in `<head>` to simulate the dark UI chrome that email clients show around the email
   - Return `$.html()`

5. **`applyDarkMode(html: string, strategy: DarkModeStrategy, originalHasDarkCss: boolean): string`**
   - If `originalHasDarkCss` is true: call `activateDarkMediaQueries(html)` then inject dark body background wrapper
   - If `originalHasDarkCss` is false: call `applyAutoInversion(html, strategy)`
   - This is the main orchestrator function called by PreviewPane
