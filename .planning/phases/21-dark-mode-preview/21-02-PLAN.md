Modify `app/components/PreviewPane.tsx` following the existing viewport toggle pattern exactly (per D-03):

1. **Add imports:**
   ```typescript
   import { hasDarkMediaQuery, applyDarkMode } from '@/lib/engine/darkMode'
   ```

2. **Add ColorMode type and state** (inside the component, after the viewport state):
   ```typescript
   type ColorMode = 'light' | 'dark'
   ```
   Move this type definition to the top of the file near the `Viewport` type.
   ```typescript
   const [colorMode, setColorMode] = useState<ColorMode>('light')
   ```
   Default is 'light' always — no localStorage read (per RESEARCH.md Pitfall 5 and Claude's discretion).

3. **Update simulatedHtml useMemo** per RESEARCH.md Pitfall 3 — check original HTML for dark CSS BEFORE applyClientRules:
   ```typescript
   const simulatedHtml = useMemo(() => {
     const originalHasDarkCss = hasDarkMediaQuery(debouncedHtml)
     const transformed = applyClientRules(debouncedHtml, ruleset)
     const withDark = colorMode === 'dark'
       ? applyDarkMode(transformed, ruleset.darkModeStrategy, originalHasDarkCss)
       : transformed
     return wrapWithSecurityHeaders(withDark)
   }, [debouncedHtml, ruleset, colorMode])
   ```
   CRITICAL: `hasDarkMediaQuery` runs on `debouncedHtml` (original), not `transformed`. This is because `applyClientRules` may strip `<style>` blocks (e.g., Naver) that contain the dark media queries.

4. **Add Light/Dark toggle buttons** in the header bar, immediately after the Desktop/Mobile toggle `</div>` and before the closing `</div>` of the right-side controls. Add a thin vertical separator between viewport and color mode toggles:
   ```tsx
   <div className="w-px h-4 bg-zinc-300 mx-1" />
   <div className="flex gap-1">
     <button
       onClick={() => setColorMode('light')}
       className={`px-2 py-0.5 text-xs rounded ${
         colorMode === 'light'
           ? 'bg-zinc-300 text-zinc-700'
           : 'text-zinc-400 hover:text-zinc-600'
       }`}
       title="Light mode"
     >
       Light
     </button>
     <button
       onClick={() => setColorMode('dark')}
       className={`px-2 py-0.5 text-xs rounded ${
         colorMode === 'dark'
           ? 'bg-zinc-700 text-zinc-200'
           : 'text-zinc-400 hover:text-zinc-600'
       }`}
       title="Dark mode"
     >
       Dark
     </button>
   </div>
   ```
   The "Dark" button uses `bg-zinc-700 text-zinc-200` for active state (visually distinct from Light's `bg-zinc-300 text-zinc-700`) to reinforce which mode is active.

5. **Update iframe background for dark mode:** When dark mode is active, the iframe container background should also be dark to simulate the email client's dark chrome:
   Change the iframe container div className from static `bg-zinc-50` to dynamic:
   ```tsx
   <div className={`flex-1 overflow-auto ${colorMode === 'dark' ? 'bg-zinc-800' : 'bg-zinc-50'} flex justify-center`}>
   ```
   And update the iframe `className` to use dynamic background:
   ```tsx
   className={`border-0 h-full ${colorMode === 'dark' ? 'bg-zinc-900' : 'bg-white'}`}
   ```
