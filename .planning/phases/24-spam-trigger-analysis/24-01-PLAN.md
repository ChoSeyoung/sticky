**Main function `analyzeSpamTriggers(html: string): SpamSummary`:**

Use `// @ts-expect-error` before `cheerio.load(html, { decodeEntities: false })`.

**Detection 1 — Excessive caps (D-01.1):**
- Extract text from body: `$('body').text()` (or just the html text if no body tag)
- Split into lines, then match words `/[a-zA-Z]{4,}/g` (minimum 4 letters per Research open Q2)
- For each word: count uppercase letters / total alpha letters. If ratio >= 0.7: push warning
- `severity: 'warning'`, `type: 'excessive-caps'`
- `fix: '대문자 사용을 줄이세요. 제목이나 강조에만 대문자를 사용하는 것이 좋습니다.'`
- Use `lineOf(html, word)` for lineNumber

**Detection 2 — Repeated punctuation (D-01.2):**
- Extract text from `$('body').text()`
- Match with regex `/([!?$]{3,})/g`
- For each match: push warning
- `severity: 'warning'`, `type: 'repeated-punctuation'`
- `fix: '반복된 특수문자를 줄이세요. 감탄부호는 하나만 사용하는 것이 좋습니다.'`

**Detection 3 — English spam keywords (D-01.4):**
- Extract text nodes only: `$('body').text()` — NOT raw html (avoids matching URLs/attributes per Research Pitfall 3)
- Convert to uppercase for case-insensitive matching
- For each keyword in SPAM_KEYWORDS_EN: if text includes the keyword, push warning
- `severity: 'warning'`, `type: 'spam-keyword-en'`
- `fix: `스팸 필터에 걸릴 수 있는 단어입니다. "${keyword}" 대신 다른 표현을 사용해 보세요.``
- `detail: { matched: keyword }`

**Detection 4 — Korean spam keywords (D-01.3):**
- Same approach as English but use SPAM_KEYWORDS_KO, substring match on `$('body').text()`
- `severity: 'warning'`, `type: 'spam-keyword-ko'`
- `fix: `스팸 필터에 걸릴 수 있는 단어입니다. "${keyword}" 사용을 줄이거나 다른 표현으로 바꿔 보세요.``

**Detection 5 — Color emphasis (D-01.5):**
- Traverse `$('[style]').each(...)` like analyzeAccessibility.ts lines 175-210
- For each element with style attribute:
  - Check `color` property for red: `/color\s*:\s*(red|#[fF]{1,2}0{1,4}|#[fF][0-9a-fA-F]0{2}00|rgb\s*\(\s*2[0-4]\d|rgb\s*\(\s*25[0-5])/i` — simplified: check if the `color` CSS property (NOT `background-color`) matches red-ish values
  - Check `font-size` >= 16px: `/font-size\s*:\s*(\d+)px/i` and parse the number, check >= 16
  - If BOTH red foreground AND large font: push warning
- `severity: 'warning'`, `type: 'color-emphasis'`
- `fix: '빨간 대형 텍스트는 스팸으로 인식될 수 있습니다. 색상이나 크기를 조절해 보세요.'`

For extracting individual CSS properties from inline style, use a helper similar to `extractCssProperty` from analyzeAccessibility.ts:
```typescript
function extractCssValue(style: string, prop: string): string | null {
  const parts = style.split(';')
  for (const part of parts) {
    const colonIdx = part.indexOf(':')
    if (colonIdx === -1) continue
    const key = part.slice(0, colonIdx).trim().toLowerCase()
    if (key === prop.toLowerCase()) {
      return part.slice(colonIdx + 1).trim()
    }
  }
  return null
}
```

Use `extractCssValue(style, 'color')` for foreground (not 'background-color') and `extractCssValue(style, 'font-size')` for size.

**Detection 6 — Image/text ratio (D-03, D-04, D-05):**
- Count images: `$('img').length`
- Extract text length: `$('body').text().replace(/\s+/g, ' ').trim().length`
- If imgCount > 0 and textLength === 0: push error with type `image-only` (D-05)
  - `fix: '이미지만으로 구성된 이메일은 스팸으로 분류될 가능성이 높습니다. 텍스트 콘텐츠를 추가하세요.'`
- Otherwise calculate ratio: `const IMAGE_WEIGHT = 200; const imageScore = imgCount * IMAGE_WEIGHT; const ratio = imageScore / (imageScore + textLength)`
- If ratio >= 0.8: push error with type `image-ratio` (D-04)
  - `fix: '이미지 비율이 너무 높습니다. 텍스트 콘텐츠를 추가하여 이미지:텍스트 비율을 40:60으로 맞추세요.'`
- Else if ratio >= 0.6: push warning with type `image-ratio` (D-04)
  - `fix: '이미지 비율이 높습니다. 텍스트 콘텐츠를 추가하여 이미지:텍스트 비율을 40:60으로 맞추세요.'`
- Store `detail: { imageRatio: Math.round(ratio * 100) / 100 }`

const errorCount = warnings.filter(w => w.severity === 'error').length
