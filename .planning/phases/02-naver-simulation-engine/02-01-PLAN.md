  <behavior>
    SIM-01 behaviors to test:

    Style block stripping:
    - Input: `<html><head><style>body{color:red}</style></head><body>hi</body></html>` + naverRuleset → Output must not contain `<style>`
    - Input: `<body><style>.foo{color:red}</style><p>hi</p></body>` + naverRuleset → Output must not contain `<style>` (body styles stripped too)
    - Input: HTML with multiple `<style>` blocks → all removed

    Inline property filtering:
    - Input: `<p style="margin: 10px; color: red;">text</p>` + naverRuleset → Output contains `color: red` but not `margin`
    - Input: `<td style="padding: 10px; font-size: 14px;">cell</td>` + naverRuleset → Output contains `font-size: 14px` but not `padding`
    - Input: `<div style="font-family: Arial; color: blue;">x</div>` + naverRuleset → Output contains `color: blue` but not `font-family`
    - Input: `<p style="MARGIN: 0; COLOR: red;">text</p>` + naverRuleset → Case-insensitive: strips MARGIN, keeps COLOR
    - Input: `<p style="margin: 0; padding: 0;">text</p>` + naverRuleset → All properties stripped, style attribute removed entirely

    Identity / passthrough:
    - Input: `<p style="color: red; font-size: 14px;">text</p>` + naverRuleset → Output preserves both properties unchanged
    - Input: HTML with no `<style>` and no blocked inline properties → Output equals input (modulo cheerio serialization)

    Entity preservation:
    - Input: `<p>Hello&nbsp;World</p>` + naverRuleset → Output contains `&nbsp;` (not `&amp;nbsp;`)

    Purity:
    - Calling applyClientRules twice with same input + naverRuleset → identical output both times

    Data-driven (per D-04):
    - Input: `<p style="margin: 0; background: red;">text</p>` + custom ruleset with strippedProperties: ['background'] → strips background, keeps margin
    - This proves engine reads from ruleset data, not hardcoded property names
  </behavior>

    ```typescript
    import * as cheerio from 'cheerio'
    import type { ClientRuleset } from '@/lib/rulesets/types'

    function filterInlineStyle(styleValue: string, blocked: ReadonlyArray<string>): string {
      const blockedSet = new Set(blocked.map(p => p.toLowerCase()))
      return styleValue
        .split(';')
        .map(s => s.trim())
        .filter(Boolean)
        .filter(decl => {
          const colonIdx = decl.indexOf(':')
          if (colonIdx === -1) return false
          const prop = decl.slice(0, colonIdx).trim().toLowerCase()
          return !blockedSet.has(prop)
        })
        .join('; ')
    }

    export function applyClientRules(html: string, ruleset: ClientRuleset): string {
      const $ = cheerio.load(html, { decodeEntities: false })

      // 1. Strip <style> elements (per D-03)
      if (ruleset.stripHeadStyles) {
        $('style').remove()
      }

      // 2. Remove blocked elements
      for (const tag of ruleset.strippedElements) {
        $(tag).remove()
      }

      // 3. Filter blocked inline properties
      if (ruleset.strippedProperties.length > 0) {
        $('[style]').each((_, el) => {
          const original = $(el).attr('style') ?? ''
          const filtered = filterInlineStyle(original, ruleset.strippedProperties)
          if (filtered) {
            $(el).attr('style', filtered)
          } else {
            $(el).removeAttr('style')
          }
        })
      }

      // 4. Allow-list enforcement (only if non-null — not active for Naver per D-04)
      // Phase 3/4 will implement this branch when allowedInlineProperties is non-null

      return $.html()
    }
    ```
