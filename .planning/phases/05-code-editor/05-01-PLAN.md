    1. Run `pnpm add @monaco-editor/react` to install Monaco Editor React wrapper (per D-01).

    2. Create directory `app/components/` if it does not exist.

    3. Create `app/components/HtmlEditor.tsx` as a Client Component:

    ```typescript
    'use client'

    import Editor from '@monaco-editor/react'

    interface HtmlEditorProps {
      value: string
      onChange: (value: string) => void
    }

    export default function HtmlEditor({ value, onChange }: HtmlEditorProps) {
      return (
        <Editor
          language="html"
          theme="vs-dark"
          value={value}
          onChange={(val) => onChange(val ?? '')}
          options={{
            wordWrap: 'on',           // per D-03: enable word wrap
            minimap: { enabled: false }, // per D-03: minimap off (email HTML is short)
            autoClosingBrackets: 'always',
            autoClosingTags: true,      // per D-03: auto-closing tags
            formatOnPaste: false,       // CRITICAL for EDIT-02: do NOT reformat on paste
            formatOnType: false,        // do not auto-format while typing
            renderWhitespace: 'selection',
            fontSize: 14,
            lineNumbers: 'on',
            scrollBeyondLastLine: false,
            automaticLayout: true,      // per D-04: auto-resize to container
            tabSize: 2,
          }}
          className="h-full"
        />
      )
    }
    ```

    Key decisions:
    - `formatOnPaste: false` is critical for EDIT-02 — prevents Monaco from reformatting pasted HTML
    - `formatOnType: false` prevents indentation mangling while typing
    - `onChange` callback passes value up to parent via props (per D-05 — state lives in parent via useState)
    - `automaticLayout: true` makes editor resize with container (per D-04 — flex layout, no fixed heights)
    - Component accepts `value` and `onChange` props so parent controls state (per D-05)
    - `theme: 'vs-dark'` is Claude's discretion — dark theme for code editing
  </action>

    1. Replace `app/page.tsx` entirely. The new page must:
       - Have `'use client'` directive (REQUIRED — Next.js 16 requires ssr: false dynamic imports to be in Client Components)
       - Use `next/dynamic` to import HtmlEditor with `{ ssr: false }` (per D-02)
       - The `dynamic()` call MUST be at module top level, NOT inside the component function
       - Use `useState<string>` to hold editor content (per D-05)
       - Provide a loading fallback while Monaco loads

    ```typescript
    'use client'

    import { useState } from 'react'
    import dynamic from 'next/dynamic'

    const HtmlEditor = dynamic(() => import('@/app/components/HtmlEditor'), {
      ssr: false,
      loading: () => (
        <div className="flex flex-1 items-center justify-center bg-zinc-900 text-zinc-500">
          Loading editor...
        </div>
      ),
    })

    const DEFAULT_HTML = `<!DOCTYPE html>
<html>
<head>
  <style>
    body { font-family: Arial, sans-serif; }
    .container { max-width: 600px; margin: 0 auto; }
  </style>
</head>
<body>
  <div class="container">
    <h1>Hello Email</h1>
    <p>Edit this HTML to preview your email.</p>
  </div>
</body>
</html>`

    export default function Home() {
      const [html, setHtml] = useState<string>(DEFAULT_HTML)

      return (
        <div className="flex flex-col h-full">
          <header className="flex items-center h-12 px-4 bg-zinc-900 border-b border-zinc-700">
            <h1 className="text-sm font-semibold text-zinc-300">Sticky — HTML Email Editor</h1>
          </header>
          <main className="flex flex-1 min-h-0">
            <HtmlEditor value={html} onChange={setHtml} />
          </main>
        </div>
      )
    }
    ```

    2. Update `app/globals.css` — add one rule to ensure the html/body/root fill viewport height. The existing `body` block already has `background` and `color`. Add `min-height: 100%` to html if not already present via Tailwind's `h-full` on the html element (which layout.tsx already sets). Check that no conflicting styles exist. The layout.tsx already has `h-full` on `<html>` and `min-h-full flex flex-col` on `<body>`, so no CSS changes should be needed. Only add if the editor doesn't fill the page.

    Key implementation notes:
    - `'use client'` on page.tsx is MANDATORY — Next.js 16 docs explicitly state `ssr: false` cannot be used in Server Components
    - `dynamic()` at module top level, not inside Home() — Next.js 16 docs require this for preload to work
    - `min-h-0` on main prevents flex overflow — without it, the editor may overflow the viewport
    - `h-full` on the outer div requires the parent chain (html > body > layout children) to all have height set — layout.tsx already does this
    - `DEFAULT_HTML` provides a starter template so users see syntax highlighting immediately
    - The `html` state variable is the contract Phase 6 will consume — it holds the complete editor content as a string
