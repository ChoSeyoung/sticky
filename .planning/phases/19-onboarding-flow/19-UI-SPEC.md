## Interaction Contract

### Onboarding Step Flow

| Step | Target Element | Popover Placement | Title | Description |
|------|---------------|-------------------|-------|-------------|
| 1 of 3 | Editor wrapper div (`ref={editorRef}`) | bottom | "HTML 이메일 에디터" | "이곳에 HTML 이메일 코드를 붙여넣거나 직접 작성하세요. 기본으로 예시 템플릿이 로드되어 있습니다." |
| 2 of 3 | Preview area wrapper div (`ref={previewRef}`) | top (or bottom if space allows) | "클라이언트별 실시간 프리뷰" | "에디터를 수정하면 오른쪽 프리뷰가 즉시 갱신됩니다. 네이버, Gmail, Outlook 등 각 클라이언트의 CSS 제한사항이 다르게 적용됩니다." |
| 3 of 3 | Client toggle bar (`ref={clientsBarRef}`) | bottom | "프리뷰할 클라이언트 선택" | "네이버는 style 블록을 제거하고, Gmail은 조건부로 제거하며, Outlook Classic은 Word 엔진을 사용합니다. 원하는 클라이언트만 선택해 비교하세요." |

### State Machine

| State | Trigger | Result |
|-------|---------|--------|
| Initial load, `sticky:onboardingCompleted` absent | App mount | `isVisible=true`, `stepIndex=0` — overlay renders |
| Initial load, `sticky:onboardingCompleted = 'true'` | App mount | `isVisible=false` — no overlay, re-entry "?" button visible |
| "다음" clicked (steps 1–2) | User click | `stepIndex++`, highlight and popover reposition |
| "완료" clicked (step 3) | User click | `localStorage.setItem('sticky:onboardingCompleted', 'true')`, `isVisible=false` |
| "건너뛰기" clicked (any step) | User click | Same as "완료" — sets localStorage, hides overlay |
| "?" button clicked (re-entry) | User click | `isVisible=true`, `stepIndex=0` |
| Window resize while overlay open | resize event | Re-measure target `getBoundingClientRect()`, update highlight/popover position |

### Accessibility

- When overlay is visible, Tab focus is trapped inside the popover card.
- Popover has `role="dialog"` and `aria-label="온보딩 가이드"`.
- Skip and Next/Done buttons are keyboard-accessible (`button` elements, no `tabIndex` manipulation needed beyond focus trap).
- ESC key triggers skip (same behavior as "건너뛰기").

## Animation Contract

| Transition | Spec | Rationale |
|------------|------|-----------|
| Overlay enter | `opacity: 0 → 1`, duration 150ms, ease-out | Subtle, non-distracting |
| Overlay exit | `opacity: 1 → 0`, duration 100ms, ease-in | Faster exit feels snappy |
| Step change (highlight reposition) | No animation — instant repositioning | Avoids flicker from async DOM measurement; use `useLayoutEffect` for sync measurement |
| Popover appear on step change | No fade — instant swap | Avoids compound animation complexity |

Implementation note: Use Tailwind `transition-opacity duration-150` for mount/unmount. Step-to-step repositioning is instant (no transition on `top`/`left` to avoid layout thrash).
